## 4. 해석 — 왜 뒤뚱거리나 (가설)
- **yaw 추종 약함**(track_ang 0.34 « track_lin 0.86, error_vel_yaw 0.58): ang_z 명령 범위(±1.57)가 커리큘럼 없이 처음부터 부과 + `heading_command=True`(heading 오차→ang_z 간접 생성)라 yaw 학습이 어려움.
- **dof_pos_limits 페널티 최대(−0.41)**: 다리 관절이 가동범위 한계 근처에서 자주 동작 → 과도한 자세/스텝.
- **명령 분포가 G1보다 공격적**: vx 2.0 m/s까지(워크-런 경계), lin_y ±0.6 → 측면/고속 명령이 뒤뚱거림(횡동요)을 유발 가능.
- **height_scan(187)이 obs에 포함**되나 flat에선 거의 상수 → 무의미한 입력 차원이 정책을 희석.
- **대칭 actor-critic + 액터가 base_lin_vel(특권) 사용** → 가치추정/정책 구조가 G1과 다름(보행 품질·sim2real에 영향 가능).
