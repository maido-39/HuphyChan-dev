## 5. 분석 (정성/정량) — ★ HOLD: 까치발(tiptoe) + 짧은 보폭
**정성 (영상)**: 로봇이 **발 전체(heel→flat→toe-off)를 안 쓰고 앞꿈치/까치발로 shuffle** (사용자 지적). 넘어지진 않고 속도추종은 되나 **비인간형**.

**정량 원인 (tensorboard reward 분해 §2b)**:
- 보상이 **속도추종에 압도**(track_ang +1.84, track_lin +0.93) → 정책이 "추종만 되는 가장 게으른 gait" 선택.
- ★ **feet_air_time = +0.0096 (사실상 꺼짐)** — threshold 0.4s 미달로 보상 미발화 → 발 안 들고 끄는 **shuffle = 짧은 보폭**.
- ★ **foot-flat / heel-toe(cop) / base-height 쉐이핑 부재** → plantigrade 강제 항 없음 → **까치발**. (gaitfix엔 있었으나 G1ImpactStable서 제거; impact 항은 -0.006로 미미해 까치발 *주범 아님*.)
- 발바닥 5캡슐은 평평(coplanar)해 까치발은 기하 강제가 아닌 **학습된 거동** = 순수 reward 문제.

**정량 (모터, `logs/measure/g1is_dm4340_flat.npz`, 새 정격 기준)** — ★ **까치발이 발목을 과부하**:
